# Chroniq V1 Stabilization Notes

## Goal

This pass stabilizes the current zero-cost video-generation foundation without changing the product direction. The focus is type safety across the API, dashboard, TTS, and Remotion render boundaries.

## Changes Made

- Added `tts_provider`, `voice_id`, and `language` to the dashboard `Video` interface so the HITL review panel matches the database video shape.
- Normalized `voiceId` in the regenerate-voice API path so `updateVideoVoiceSettings` receives `string | null`, not `undefined`.
- Made Edge TTS TLS socket data handling explicitly convert incoming chunks to `Buffer` before concatenation and WebSocket frame parsing.
- Typed the Remotion render progress callback to remove the implicit `any`.
- Cast the Remotion composition component at the registration boundary so Remotion's loose prop type accepts the strongly typed video component.
- Preserved the current Chromium launch options and cast that Remotion option object because Remotion 4.0.125's public type does not expose the custom `args` field used by this renderer.

## Current Intended V1 Flow

1. Creator chooses topic, format, language, TTS provider, and voice.
2. API creates a video record with those generation settings.
3. Worker generates script, voice, captions, scenes, thumbnail, and metadata.
4. Video pauses at `awaiting_approval`.
5. Creator reviews script/audio/scenes, can regenerate voice/captions or replace scene images.
6. Approval queues Remotion render and produces `final.mp4`.
